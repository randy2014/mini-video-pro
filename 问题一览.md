# 问题一览

> 迷你视频项目开发过程中踩过的坑与修复记录，按时间排列。

---

## 1. 真·源码位置陷阱（双层嵌套）

| 项 | 说明 |
|----|------|
| 症状 | 改了文件、提交 push，线上毫无变化；grep 查不到预期内容 |
| 根因 | git 仓库根 `/workspace/video-entitlement-server/` 里还有一个同名 `video-entitlement-server/` 子目录，**真正的 Spring Boot 源码在第二层**。直接对第一层 `src/` 改/查 → 落到空壳。 |
| 真实路径 | 后端：`video-entitlement-server/video-entitlement-server/src/main/java/com/video/entitlement/module/...` <br> 前端：`video-entitlement-admin/src/...`（仓库根直接挂） |
| 修复 | 永远以 `git ls-files` 确认真实路径，不改 `/workspace/video-entitlement-admin/`（那是独立 clone 副本，不参与部署） |

---

## 2. 日期格式遗漏

| 项 | 说明 |
|----|------|
| 症状 | 用户管理页权益到期时间、仪表盘日志时间仍显示原始 ISO 串（`2026-07-31T00:00`） |
| 根因 | 全局日期格式化提交 `b229d34` 遗漏了两处：UserManage 的 `expireTime` 和 Dashboard 的 `createdAt`（此前误改到了独立 clone 目录 `/workspace/video-entitlement-admin/`） |
| 修复 | 真正源码（仓库内 `video-entitlement-admin/`）补 `fmt()` → `dayjs(s).format('YYYY-MM-DD HH:mm')` |

---

## 3. AppVersionController 路径 bug（预存）

| 项 | 说明 |
|----|------|
| 症状 | 管理端版本列表等接口一直返回 `10500 NoResourceFoundException`，路由 404 |
| 根因 | 类级 `@RequestMapping("/api/v1")` + 方法 `@GetMapping("/admin/versions")` → 实际解析为 `/api/v1/admin/versions`，但安全规则和前端都期望 `/admin/api/v1/versions` |
| 修复 | 每个管理端方法**写全路径** `/admin/api/v1/versions`，客户端接口保留 `/api/v1/client/version` |

---

## 4. Spring 上传默认 1MB 上限

| 项 | 说明 |
|----|------|
| 症状 | 上传 APK（5–7MB）报 `MaxUploadSizeExceededException` |
| 根因 | Spring 默认 `max-file-size=1MB`，且第一次加配置的编辑**未落盘**（提交后文件里没有） |
| 修复 | `application.yml` 加 `spring.servlet.multipart.max-file-size: 50MB` / `max-request-size: 60MB`；**提交前务必 `grep` 确认存在** |

---

## 5. 上传落盘必须给后端挂卷

| 项 | 说明 |
|----|------|
| 症状 | 管理端上传 APK 成功但文件不存在 |
| 根因 | 后端容器 **没有挂载 `/data/video-apk`**，`storeFile()` 写的是容器内临时路径 |
| 修复 | `deploy.yml` 后端 `docker run` 加 `-v /data/video-apk:/data/video-apk`（RW）及 `APP_APK_DIR` / `APP_DOWNLOAD_BASE` 环境变量 |

---

## 6. Android CI 上传必须用 root（非 ubuntu）

| 项 | 说明 |
|----|------|
| 症状 | `build-android.yml` 的 scp 上传报 `Permission denied` 写 `/data/video-apk/` |
| 根因 | `/data/video-apk` 属主 `root:root`、权限 `755`，**只有 root 能写**。CI 工作流默认用 `ubuntu` 用户 → 无写权限。原 `build_and_deploy.sh` 用 `root@host`，所以本地能传。 |
| 修复 | 工作流 `env.SERVER_USER` 改为 **`root`**（不动服务器、不加密钥，与原脚本一致） |

---

## 7. CI 构建完成后解析下载地址踩坑

| 项 | 说明 |
|----|------|
| 症状 1 | `gh run view --log | grep DOWNLOAD_URL` 抓到了 `echo` 脚本行里**未展开的 `${APK_NAME}` 字面量**，导致登记到版本表的下载地址为 `/downloads/${APK_NAME}` |
| 症状 2 | SSH `ls /data/video-apk/ | grep 版本 | head -1` 按字母序抓到了 `qr-*.png`（二维码）而非 `.apk` |
| 修复 | 取真实文件名：`ls /data/video-apk/ | grep 202607201418 | grep -E '\.apk$'`；用 `-E '\.apk$'` 精确匹配 |

---

## 8. 平台编辑时 platformCode 不提交（Ant Design disabled）

| 项 | 说明 |
|----|------|
| 症状 | 管理后台编辑平台并输入 logo 链接后报 `10002 "platformCode: must not be blank"`（后端 `@NotBlank` 校验失败） |
| 根因 | `PlatformManage.tsx` 编辑模式下 `platformCode` 输入框 `disabled={!!editingId}`，Ant Design 的 disabled 字段**不随表单提交** → 请求体缺 `platformCode` → 后端校验报错 |
| 修复 | `handleSubmit` 里加 `form.getFieldValue('platformCode')` 补回（Ant Design 的 `getFieldValue` 可从表单 store 读取 disabled 字段的值） |

---

## 9. 明文凭据泄露

| 项 | 位置 | 风险 | 状态 |
|----|------|------|------|
| git remote URL 含明文 `ghp_***` PAT | `git remote -v` | 任何人拿到 PAT 可读写仓库 | ⚠️ 待吊销重发 |
| `README.md` 含明文 keystore 密码 `MiniVideo@2026!` | 仓库根 `README.md` / `ANDROID_SIGNING.md` | 仓库公开时泄露签名密钥密码 | ⚠️ 仓库已转私有，短期可接受 |
| `PROJECT_MEMORY.md` 含明文 DB/Redis/Admin 口令 | `/workspace/PROJECT_MEMORY.md` | 提交到 git 后历史留痕 | ⚠️ 仓库已转私有 |

> 原则 #3：**明文密钥不写进文件**，仅记录"是什么、在哪"，密码存入 GitHub Actions secrets。
